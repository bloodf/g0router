# Waves 0–3 correctness audit (Fable 5, 2026-06-12)

Method: targeted extraction across the merged tree + matrix cross-check
(`matrix/9router-routing.md` §g0router notes independently corroborate). Verdicts
are mine; every claim carries file:line evidence.

## Overall verdict

The Wave 1–3 LOGIC is sound: full suite + `go test -race` green across all merges;
the diff gates caught and we fixed 2 security holes (OIDC probe stored-secret leak;
CLI-token /v1 bypass) and ~10 real defects pre-merge. Wave 1 translators are
exercised end-to-end by Wave-2 adapters and their tests — no latent translation
issues surfaced downstream. HOWEVER, the audit found that two Wave-3 features are
implemented and tested but NOT WIRED at the composition root, plus three small real
bugs. None invalidate the waves; all are contained and scheduled in `w4-pre`.

## HIGH — wiring gaps (features exist, production never uses them)

- **G1: Credential resolution never reaches the request path.**
  `Router.SetKeyResolver` (`internal/inference/router.go:36`) has ZERO production
  callers; `auth.NewCredentialResolver` (`internal/auth/credentials.go:29`) is never
  constructed in `internal/server/*` or `cmd/*`. Consequently `Resolve` returns
  empty-value keys and every live upstream call fails auth unless a key arrives some
  other way — `internal/api/chat.go:63` and `embeddings.go:37` still read "Phase 6+
  will wire the key store". The w3-f resolver (refresh, single-flight, psd) is fully
  tested — including the round-3 store-backed integration test — but only tests wire
  it. The matrix knew: "Router returns empty key values" (9router-routing.md §g0router).
- **G2: gemini/xai OAuth flows unreachable in production.**
  `internal/server/server.go:36-38` builds `flows = {"anthropic": ...}` only;
  `internal/admin/oauth.go:21,53,141` does `h.flows[providerType]` → gemini/xai
  start/callback return not-found live, despite handlers + passing tests (tests build
  their own flows map). Same flows map gap also starves the G1 resolver's refresh.

## MEDIUM — real bugs

- **G3: `GET /v1/models/{id}` ignores the id** — `internal/api/models.go:57-60`
  `Get` delegates to `List` unfiltered (comment admits "Phase 4"). API contract bug.
- **G4: silent deterministic fallback in key-id generation** —
  `internal/auth/apikey.go:183-189` `randomUUID` returns `"0000000000000000"` when
  crypto/rand fails instead of propagating the error (generator signature already
  returns error). Violates errors-are-values; in the worst case mints a predictable
  key segment.
- **G5: stream loop has no client-abort check** — `internal/api/chat.go` (and
  messages/responses) range over the provider channel without selecting on request
  cancellation (matrix §g0router). Goroutine + upstream work leak on client
  disconnect. Contained fix (fasthttp `ctx.Done()`), scheduled in w4-pre; the fuller
  abort/retry interplay is w4-b.

## LOW — hygiene

- **G6: stale phase comments contradict merged code** — `internal/inference/
  router.go:17-18,43,45` still say "minimal Phase 5 / TODO(phase-8)" although w2-d
  replaced the router; `chat.go:63`/`embeddings.go:37` "Phase 6+" resolves with G1.

## Explicitly NOT bugs (verified wired)

- Guard validators ARE injected in production (`server.go:43-46`). Ollama host
  override IS consumed (`ollama/chat.go:24,38,172` via `key.ProviderSpecificData`).
  `translation.NewRegistry()` registers all pairs internally (`registry.go:138+`).
  `/v1/models` aggregation is routed (`routes_openai.go:21`). SeedAdmin seeds
  admin/"123456" at boot (`cmd/g0router/main.go:27-28,110`) — matches the ref default;
  empty-hash path arises only via `reset-password`, as designed.
- Matrix-noted absences that are PLANNED waves, not defects: no error-classification
  layer, no connection status columns, no structured strategies, no virtual-key table
  (all Wave-4/5 scope); `bypass_handler.go` ported in w1 but unwired = PAR-ROUTE-034
  MISSING, scheduled w4-f.

## Disposition

G1–G6 are bundled into `plans/w4-pre-wiring-fixes.md` (with the Wave-1 deferral
PAR-TRANS-006/051/052/053 helpers), gated and dispatched FIRST in Wave 4 — G1/G2 are
prerequisites for any routing work that assumes real credentials.
