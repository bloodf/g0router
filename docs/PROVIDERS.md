# Provider Matrix

`internal/provider/matrix.go` is the source of truth for provider status. Public APIs and CLI provider lists must derive from that matrix instead of maintaining separate handwritten lists.

Status meanings:

- `supported`: public inference works through normal `g0router serve` routing today.
- `adapter_only`: an adapter is registered in startup, but public support is still limited by missing model catalog entries, provider capability routing, authentication, streaming, tooling, or other documented gaps.
- `auth_only`: credential capture exists, but no inference adapter is wired.
- `unsupported`: explicitly not implemented; do not advertise as usable.

Current public direct-dispatch providers are `openai`, `alibaba`, `anthropic`, `azure`, `bedrock`, `cerebras`, `cloudflare-ai-gateway`, `cohere`, `deepseek`, `fireworks`, `gemini`, `github-copilot`, `gitlab-duo`, `groq`, `huggingface`, `kilo`, `kimi`, `litellm`, `lm-studio`, `mistral`, `minimax`, `nebius`, `nvidia`, `ollama`, `ollama-cloud`, `opencode`, `openrouter`, `perplexity`, `qianfan`, `qwen`, `replicate`, `together`, `vercel-ai-gateway`, `vertex`, `vllm`, `xai`, `xiaomi`, and `zhipu`. Providers without a static catalog use provider-qualified model names such as `alibaba/<model>`, `azure/<deployment>`, `cloudflare-ai-gateway/<author>/<model>`, `github-copilot/<model>`, `gitlab-duo/<model>`, `kilo/<model>`, `kimi/<model>`, `ollama-cloud/<model>`, `opencode/<model>`, `qianfan/<model>`, `replicate/<owner>/<model>`, `vllm/<served-model>`, `xiaomi/<model>`, or `zhipu/<model>`.

## Public Surfaces

- `GET /api/providers` returns the full matrix with status and capability fields.
- `g0router providers list` prints only public direct-dispatch providers.
- `g0router auth list` prints providers with an auth flow, including `auth_only` providers.
- `/api/connections` lists stored credentials for every provider, including auth-only rows, but that does not imply inference support.

## Model Routing Caveat

Current dispatch resolves stored model aliases first, then exact model catalog matches, then provider-qualified dynamic routes for deployment-defined or provider-owned model IDs, then legacy `gpt-*` and `claude-*` prefixes. Exact catalog matches provide public routing for OpenAI, Anthropic, Bedrock, Cerebras, Cohere, DeepSeek, Fireworks, Gemini, Groq, Hugging Face, Mistral, MiniMax, Nebius, NVIDIA, Ollama, OpenRouter, Perplexity, Qwen, Together, Vercel AI Gateway, Vertex, and xAI model IDs when a matching active connection exists. Provider-qualified dynamic routes strip the provider prefix before upstream dispatch for Alibaba, Azure, Cloudflare AI Gateway, GitHub Copilot, GitLab Duo, Kilo, Kimi, LiteLLM, LM Studio, Ollama Cloud, OpenCode Zen, Qianfan, Replicate, vLLM, Xiaomi, and Zhipu, so `azure/gpt-4o-prod` reaches Azure as deployment `gpt-4o-prod`, `alibaba/qwen3-max-2026-01-23` reaches DashScope as `qwen3-max-2026-01-23`, `cloudflare-ai-gateway/openai/gpt-4.1` reaches Cloudflare as `openai/gpt-4.1`, `gitlab-duo/duo-chat-gpt-5-1` reaches GitLab AI Gateway's OpenAI proxy as `gpt-5.1-2025-11-13`, `gitlab-duo/duo-chat-sonnet-4-5` reaches GitLab AI Gateway's Anthropic proxy as `claude-sonnet-4-5-20250929`, `kilo/anthropic/claude-sonnet-4.5` reaches Kilo Gateway as `anthropic/claude-sonnet-4.5`, `kimi/kimi-k2.6` reaches Moonshot as `kimi-k2.6`, `ollama-cloud/gpt-oss:120b` reaches native Ollama Cloud as `gpt-oss:120b`, `opencode/anthropic/claude-sonnet-4` reaches OpenCode Zen as `anthropic/claude-sonnet-4`, `qianfan/deepseek-v3.1-250821` reaches Qianfan as `deepseek-v3.1-250821`, `replicate/owner/model` reaches Replicate prediction creation as model `owner/model`, `xiaomi/claude-sonnet-4` reaches Xiaomi as `claude-sonnet-4`, and `zhipu/glm-5.1` reaches Z.AI as `glm-5.1` without adding fake static pricing. Bedrock catalog routing is limited to documented non-streaming Converse models such as `anthropic.claude-3-5-haiku-20241022-v1:0`. Vertex catalog routes are provider-qualified (`vertex/gemini-*`) and are rewritten to upstream Gemini model IDs before dispatch so unqualified `gemini-*` remains owned by the Gemini adapter. Explicit aliases can target registered adapter providers only when the provider matrix marks inference capability true, `combo/*` routes use the same dispatch path, and request logging uses dispatch metadata when available. OpenRouter is quota-capable through `GET /api/v1/key` using the stored API key and returns credit units, including explicit unlimited state when upstream omits a finite limit. Quota fetchers are intentionally unsupported for providers whose matrix `quota` field is `false`; those routes fail open rather than fabricating provider quota data.
