## Section 2: Shape Mismatches

### 2.1 CRITICAL — Response envelope mismatch (affects every endpoint)

The UI `apiFetch` client (`ui/src/lib/api/client.ts`) expects **every** response to be wrapped in `{data, error}`:

```ts
const json = await response.json();
if (json.error !== null && json.error !== undefined) {
  throw new ApiError(response.status, json.error);
}
return json.data as T;
```

The backend is **inconsistent**:

- **Some** endpoints wrap lists in `listResponse[T]{Data: [...]}` (e.g. `GET /api/connections`, `GET /api/keys`, `GET /api/audit`). These happen to work because `json.data` exists.
- **Many** endpoints return **raw objects** with no `data` field, causing `apiFetch` to return `undefined`:
  - `GET /api/settings` → raw `store.Settings`
  - `GET /api/providers/:id` → raw `providerDetailResponse`
  - `POST /api/keys` → `createAPIKeyResponse{Key: ..., Raw: ...}`
  - `PUT /api/keys/:id` → `createAPIKeyResponse{Key: ...}`
  - `POST /api/connections` → raw `connectionResponse`
  - `PUT /api/connections/:id` → raw `connectionResponse`
  - `GET /api/usage/summary` → raw `usageSummaryResponse`
  - `GET /api/usage/chart` → raw `usageChartResponse`
  - `POST /api/connections/:id/test` → raw test result
  - `POST /api/providers/test-batch` → raw batch result
  - `GET /api/auth/status` → raw `authStatusResponse` (actually backend wraps this one in `{"data": ...}`)

**Impact:** When the UI connects to the real backend, most pages will receive `undefined` data and silently fail or crash.

## Section 4: Recommendations

### Immediate (blocks basic functionality)

1. **Fix the response envelope mismatch**
   - **Option A (recommended):** Update `apiFetch` to handle both wrapped and unwrapped responses. Add a compatibility layer that checks for `.data` and falls back to the root object:
     ```ts
     return json.data !== undefined ? json.data : json;
     ```
   - **Option B:** Update every backend handler to consistently wrap responses in `{"data": ...}` and errors in `{"error": ...}`. This is a large, risky change.

2. **Fix list response shapes for `/api/usage` and `/api/audit`**
   - **Option A:** Change backend to return `{items, total}` instead of `{object, data, limit, offset, total}`.
   - **Option B:** Add a `listSelector` prop to the components (or update `apiFetch` wrappers) to extract `json.data` as `items`.

3. **Add missing CRITICAL endpoints**
   - `GET /api/models` — Aggregated model catalog. Can delegate to `handlers.Models` or a new admin wrapper that returns the same data in a simpler format.
   - `POST /api/keys/:id/regenerate` — Add to `api/handlers/apikeys.go`. Should generate a new secret, update the key, and return the new view.
   - `GET /api/quota` — Aggregate quota. Option A: new backend handler that iterates providers and calls each quota fetcher. Option B: adapt UI to make multiple calls to `/api/usage/quota/:provider`.

### Short-term (needed for full feature parity)

4. **Align `/api/audit` field names**
   - Backend should return `actor` (human-readable username) alongside or instead of `actor_api_key_id`, or the UI should be adapted to display `actor_api_key_id`.
   - Backend `id` is `int64`; UI expects `string`. Ensure JSON serialization casts to string.

5. **Align `/api/usage` fields**
   - Add `prompt_tokens` / `completion_tokens` aliases (or rename in UI) to match `input_tokens` / `output_tokens`.
   - Add a computed `status` field (`"success"` / `"error"`) based on status code, or update UI to derive it client-side.
   - Decide on nullability: either backend guarantees non-null for core fields, or UI types should accept `number \| null`.

6. **Align `/api/settings` fields**
   - Add `theme` and `language` to `store.Settings` (backend), or remove from UI if not supported.
   - Remove `tunnel_dashboard_access` from UI mock if not a real feature.

7. **Fix `/api/pricing` URL pattern**
   - Either change backend from `/api/pricing/:provider/:model` to `/api/pricing/:id` (using a composite ID like `"provider/model"`), or update `CrudPage` / UI to construct two-segment URLs.

8. **Align `/api/connections` fields**
   - Add `models: string[]` to backend `connectionResponse`, or remove from UI if connections no longer expose selected models.

10. **Diagnostics endpoint** — Same as above; page is `ComingSoon`.

## Summary of Most Critical Gaps

1. **`apiFetch` response envelope** — The UI client expects `{"data": ...}` for every response, but the backend often returns raw objects. **This will break almost every page** when the UI connects to the real backend.
2. **`GET /api/models` missing** — The Models page and Combos page both call this. No backend equivalent exists.
3. **`POST /api/keys/:id/regenerate` missing** — The Endpoint page has a working "Regenerate" button that will 404 against the backend.
4. **`/api/usage` and `/api/audit` list shapes** — UI expects `{items, total}`; backend returns `{data, total, limit, offset, object}`. Logs and Audit pages will show empty tables.
5. **`GET /api/quota` missing** — The Quota Tracker expects an aggregate list; backend only supports per-provider quota.
